TWLKR-TMS 대시보드 상세 명세서
개요
TWLKR-TMS(Teckwah Logistics Korea - Transportation Management System)는 물류 관리를 위한 대시보드 시스템으로, 배송 주문 관리, 인수인계, 데이터 시각화 기능을 제공합니다. 시스템은 총 3개의 주요 페이지(TMS, Work-Notice, Visualization)로 구성되어 있으며, 사용자 친화적인 인터페이스를 통해 효율적인 업무 관리를 지원합니다.
1. 공통 UI 구성요소
1.1 사이드바

로고 영역: TWLKR-TMS 로고 및 시스템명 표시
사용자 정보: 사용자 아바타, 이름, 역할(CS, HES, LENOVO 등) 표시
메뉴: 3개의 주요 메뉴 항목 제공

TMS (기본 대시보드)
Work-Notice (업무 공지 및 인수인계)
Visualization (데이터 시각화)


로그아웃 버튼: 시스템 로그아웃 기능

1.2 메시지 팝업

시스템 알림을 표시하는 팝업 컴포넌트
성공, 오류, 경고, 정보의 4가지 유형 지원
하단에서 상단으로 자연스럽게 페이드인 애니메이션 적용
일정 시간 후 자동으로 사라짐

1.3 모달 시스템

다양한 작업(상세보기, 신규등록, 수정, 상태변경 등)을 위한 모달 다이얼로그
반투명 오버레이와 중앙 배치된 모달 컨텐츠로 구성
반응형 디자인으로 다양한 화면 크기에 대응

2. TMS 페이지 (index.html)
2.1 기본 레이아웃

날짜 필터 영역: ETA 기간 설정 및 조회 기능

시작일/종료일 선택 입력
조회 및 Today 버튼


검색 영역: 주문번호 기반 빠른 검색 기능
필터 패널: 상태, 부서, 창고별 필터링 기능
요약 카드: 중요 지표를 한눈에 볼 수 있는 6개의 카드

배송 건수, 회수 건수, 진행 건수, 완료 건수, 대기 건수, 기타(취소+이슈) 건수


액션 버튼: 데이터 관리를 위한 기능 버튼(새로고침, 상태변경, 배차처리, 신규등록, 삭제)
데이터 테이블: 페이지네이션이 적용된 주문 데이터 표시

체크박스 선택 기능
상태별 행 색상 구분
주문번호 복사 기능



2.2 주요 기능
2.2.1 주문 관리

조회: 날짜 및 필터 기반 조회 기능
등록: 새로운 배송/회수 주문 등록
수정: 기존 주문 정보 수정
삭제: 선택한 주문 삭제

2.2.2 상태 관리

상태 변경: 선택한 주문의 상태 일괄 변경 기능

대기, 진행, 완료, 이슈, 취소 상태로 변경 가능
상태 변경 시 자동으로 시간 기록(진행 상태 → 출발 시간, 완료/이슈 → 완료 시간)



2.2.3 배차 관리

배차 처리: 배송기사 정보 할당 기능

기사 이름 및 연락처 정보 입력
배차 후 상태 자동 변경 없음(상태는 별도 관리)



2.2.4 정보 조회

상세 정보: 주문 항목 클릭 시 상세 정보 모달 표시

기본 정보: 주문번호, 상태, 고객, 유형, 부서, 창고, SLA, 우편번호
배송 정보: ETA, 생성시간, 출발시간, 완료시간, 배송기사, 주소, 연락처
메모 정보: 주문 관련 메모
업데이트 정보: 최종 수정 시간 및 담당자



2.3 디자인 특징

상태별 색상 코드: 직관적인 상태 파악을 위한 색상 시스템

대기(황색), 진행(파란색), 완료(녹색), 이슈(빨간색), 취소(회색)


반응형 레이아웃: 다양한 화면 크기에 맞춰 조정되는 UI

모바일 화면에서 사이드바 축소 및 컨텐츠 재배치
테이블 스크롤 지원으로 모든 데이터 접근성 확보


로딩 인디케이터: 데이터 로딩 중 상태 표시
그림자 효과: 카드 및 버튼에 미묘한 그림자로 깊이감 부여

3. Work-Notice 페이지 (handover.html)
3.1 기본 레이아웃

필터 영역: 공지사항/인수인계 표시 여부 선택 체크박스
액션 버튼: 새로고침 및 신규 등록 버튼
분할 화면: 두 개의 섹션으로 분할된 화면 구성

공지사항 섹션(상단 30%)
인수인계 섹션(하단 70%)



3.2 주요 기능
3.2.1 공지사항/인수인계 관리

조회: 작성자, 작성일시, 제목, 내용 정보 표시
섹션 표시/숨김: 체크박스로 공지사항/인수인계 섹션 개별 표시 제어
상세 보기: 항목 클릭 시 상세 정보 모달 표시
등록: 새로운 공지사항/인수인계 등록

제목, 내용 입력 및 공지사항 여부 설정


수정: 기존 항목 수정(본인 작성 항목만 가능)
삭제: 항목 삭제(본인 작성 항목만 가능)

3.2.2 사용자 권한 제어

권한 관리: 작성자와 현재 사용자 비교로 수정/삭제 권한 제어

본인이 작성한 항목만 수정/삭제 버튼 표시
타인이 작성한 항목은 조회만 가능



3.3 디자인 특징

분리된 섹션: 명확한 구분을 위한 섹션 분리 디자인
텍스트 처리: 길이가 긴 내용의 자연스러운 처리

제목/내용 필드에 말줄임표(...)로 텍스트 오버플로우 처리
상세 보기에서 전체 내용 확인 가능


테이블 고정 너비: 각 컬럼별 너비 고정으로 일관된 레이아웃 유지
페이지네이션: 각 섹션별 독립적인 페이지네이션 제공

4. Visualization 페이지 (visualization.html)
4.1 기본 레이아웃

컨트롤 패널: 시각화 설정을 위한 단계별 인터페이스

1단계: 시각화 유형 선택
2단계: 필터 설정(기간, 부서 등)


차트 영역: 선택한 시각화 유형에 따른 차트 표시

기본 차트 컨테이너
부서별 차트 그룹



4.2 주요 기능
4.2.1 시각화 유형

시간대별 주문 접수: 시간대별 부서별 주문 건수 시각화

시간대 구분: 9시18시(1시간 단위), 18시20시, 20시00시, 00시9시(각각 하나의 단위)
부서별(CS, HES, LENOVO) 데이터 분리 표시
그룹화된 막대 차트로 시간대별 비교 가능


부서별 배송 상태 분포: 부서별 배송 상태 비율 시각화

3개 부서(CS, HES, LENOVO)별 개별 파이 차트
상태별(대기, 진행, 완료, 이슈, 취소) 비율 시각화
각 부서의 상태 분포 직관적 비교 가능



4.2.2 데이터 필터링

날짜 필터: 선택한 시각화 유형에 따라 다른 기준 적용

시간대별 차트: 접수일(create_time) 기준
부서별 차트: ETA 기준


부서 필터: 특정 부서 데이터만 표시 옵션(사용되지 않음)

4.3 디자인 특징

단계적 인터페이스: 사용자 경험 개선을 위한 단계별 UI

명확한 단계 표시(번호, 제목)로 사용 흐름 안내


로딩 인디케이터: 차트 로딩 중 상태 표시
차트 스타일: 직관적인 데이터 인식을 위한 차트 디자인

막대 차트: 부서별 색상 구분, 간격 및 레이블 최적화
파이 차트: 상태별 색상 코드, 퍼센트 및 수치 표시


빈 데이터 처리: 데이터가 없는 경우 적절한 안내 메시지 표시
