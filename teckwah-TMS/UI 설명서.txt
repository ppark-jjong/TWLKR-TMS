# 프론트 디자인 상세 설명서

본 문서는 **기존 동작 설명서**의 기능 요구사항을 반영하며, **UI/UX 및 레이아웃** 관점에서 프론트엔드 구현 시 필요한 세부 사항을 정의합니다.  
디자인은 **Pretendard 폰트**를 전체적으로 사용하고, **빅테크 스타일의 깔끔하고 심플한 화이트톤**을 기반으로 하며, **바로 서비스가 가능**하도록 구성합니다.

---

## 0. 공통 레이아웃 및 디자인

1. **사이드바 (Sidebar)**
   - **열고 접기**가 가능한 형태 (예: GPT 디자인처럼 Collapse/Expand 기능)
   - **상단 영역**: 계정 정보(프로필 사진/이름/아이디 등)와 각 페이지로 이동할 수 있는 메뉴
   - **하단 영역**: 로그아웃 버튼과 **브랜드 로고** (사이드바 제일 아래)
   - 전체 레이아웃에서 사이드바가 펼쳐져 있을 때 페이지 목록과 계정 정보가 명확히 보이고, 접었을 때는 아이콘/툴팁만 표시
   - 사이드바 너비는 접었을 때 최소화, 펼쳤을 때 여유 있는 공간 유지

2. **폰트**
   - 전역적으로 **Pretendard** 폰트를 사용
   - 가능한 시스템 폰트 대체가 되도록 `font-family`는 `"Pretendard", -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, ...` 식으로 설정

3. **전체 테마**
   - 깔끔한 **화이트톤** 배경
   - 적절한 **그레이 음영**을 사용해 섹션 구분
   - **빅테크(대기업 웹 서비스)** 느낌을 살려, 여백과 간격을 충분히 두어 시인성을 높임
   - 버튼, 드롭다운, 모달 등 모든 컴포넌트는 **미니멀**하고 **직관적인** 스타일

4. **반응형 디자인**
   - 웹 브라우저 너비가 작아질 경우 사이드바 자동 접기 또는 숨김
   - 주요 테이블, 카드, 모달은 모바일/태블릿 환경에서도 UI가 깨지지 않도록 유연하게 구성

## 1. 대시보드 페이지 (DashboardPage)

### 1-1. 상단 고정 영역
1. **날짜 기간 선택 (Date Range Picker)**
   - 기본값: 오늘 날짜
   - 변경 시 해당 기간 데이터 재조회

2. **검색바**
   - 주문번호(order_no) 검색 전용
   - 검색 버튼 클릭 시 서버로 검색 요청

3. **필터링 (CSR)**
   - **부서(Department)**, **창고(Warehouse)**, **상태(Status)** 각각 드롭다운
   - 이미 로드된 데이터에 대해 클라이언트 측에서 필터링

4. **버튼들**
   - **새로고침**: 대시보드 데이터를 다시 로드  
   - **생성**: 새 주문 생성 모달창 열기  
   - **배차**: 기사 배정(배차) 모달창 열기  
   - **상태변경**: 상태 변경 모달창 열기  
   - **삭제**: 선택된 주문들 삭제

5. **테이블 컬럼 커스터마이징 (CSR)**
   - 표시할 컬럼 선택 및 순서 변경 가능
   - 사용자 설정은 브라우저 로컬스토리지 등에 저장해 유지

> **상단 고정**  
> 날짜 기간 선택, 검색바, 필터링, 버튼들, 테이블 컬럼 커스터마이징 옵션은 화면 스크롤에 영향을 받지 않도록 고정

---

### 1-2. 테이블 리스트 영역
- **표시 컬럼 순서**:  
  `type`, `department`, `warehouse`, `order_no`, `sla`, `eta`, `region`, `customer`, `driver_name`
- **행 선택**: 체크박스로 단일/다중 행 선택
- **행 클릭**: 상세 정보 모달 열림
- 스크롤 시에도 상단 고정 영역은 그대로 유지

---

### 1-3. 상세 정보 모달창
- 해당 주문의 **모든 정보** 표시
- **수정 가능 항목**:  
  `order_no`, `type`, `department`, `warehouse`, `SLA`, `eta`, `postal_code`, `address`, `customer`, `contact`, `remark`
- 수정 후 **저장** 시 서버로 업데이트 요청, **취소** 시 변경 내용 폐기

---

### 1-4. 주문 생성 모달창
- **입력/선택 필드**:
  1. `order_no` (input)
  2. `type` (드롭다운)
  3. `department` (드롭다운)
  4. `warehouse` (드롭다운)
  5. `SLA` (input)
  6. `eta` (datetime picker)
  7. `postal_code`
  8. `address`
  9. `customer`
  10. `contact`
  11. `remark`
- **생성** 버튼 클릭 시 생성 요청
- **취소** 버튼 클릭 시 모달 닫힘

---

### 1-5. 배차(기사 배정) 모달창
- **driver_name** (input)
- **driver_contact** (input)
- **저장** 버튼 클릭 시 선택된 주문들에 기사 정보 일괄 배정
- **취소** 버튼 클릭 시 모달 닫힘

---

### 1-6. 상태 변경 모달창
- **상태** 드롭다운 (대기, 진행, 완료, 이슈, 취소)
- **저장** 버튼 클릭 시 선택된 주문들에 상태 일괄 반영
- **취소** 버튼 클릭 시 모달 닫힘

---

### 1-7. 주문 삭제
- 체크박스로 단일/다중 선택 후 **삭제** 버튼 클릭
- **USER 권한**: 본인 생성 주문만 삭제 가능  
- **ADMIN 권한**: 모든 주문 삭제 가능

---

## 2. 인수인계 페이지 (HandoverPage)

### 2-1. 페이지 레이아웃 (3:7 구조)
- **상단/좌측 3**: 공지사항(Notice) 목록
- **하단/우측 7**: 인수인계(Handover) 목록
- 두 목록 모두 독립적으로 **페이지네이션** 적용 가능

### 2-2. 공지사항 섹션
- **공지사항 목록** 표시 (카드 또는 테이블 형태)
- 새 공지사항 등록 시, 인수인계와 동일한 모달창을 사용하되 “공지사항 등록” 체크박스가 체크됨
- **삭제**: USER는 본인 작성만, ADMIN은 모든 항목

### 2-3. 인수인계 섹션
- **인수인계 목록** 표시 (카드 or 테이블)
- **작성**: 새 인수인계 등록 모달
- **삭제**: USER는 본인 작성만, ADMIN은 모든 항목
- **수정 불가**: 기존 인수인계는 재등록 방식으로만 변경

---

## 3. 요약

- 사이드바(열고 접기 가능) + 상단 고정 영역(날짜/검색/필터/버튼), 테이블/모달 구조
- **Pretendard** 폰트를 전체적으로 사용하고, **화이트톤** + **빅테크 스타일** 디자인
- **대시보드**는 날짜 범위, 검색, 필터, CRUD, 상태/기사 배정 모달 중심
- **인수인계**는 공지사항과 인수인계를 한 페이지에서 (3:7 레이아웃) 모두 확인

이로써 **바로 서비스가 가능한 깔끔하고 직관적인 프론트 디자인**을 구현할 수 있습니다.
