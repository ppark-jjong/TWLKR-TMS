
이번에는 [주문 상세정보 모달] 기능을 구현하려고 합니다. 
이 기능은 다음 조건을 충족해야 합니다:
1. 대시보드 페이지에서 주문 행을 클릭하면 모달이 열림
2. API(/dashboard/orders/{id})를 통해 상세정보 로드
3. 락이 걸려있으면 수정 버튼 비활성화
4. 모든 주문 필드 및 마지막 업데이트 정보 표시
구현 시 SSR/CSR 경계를 명확히 하고 코드를 단순하게 유지하세요.

현재 대시보드 기능의 코드 성능을 개선하고 싶습니다.
특히 다음 부분을 집중적으로 검토해주세요:
1. API 호출 최적화 (중복 요청 방지)
2. 이벤트 위임 패턴 적용 (개별 행에 이벤트 핸들러 추가 대신)
3. 불필요한 DOM 조작 최소화
코드 구조는 유지하면서 성능 개선 방안을 제안해주세요.


대시보드 페이지에서 다음 버그가 발생합니다:
1. 상태 필터 적용 후 다른 필터 적용 시 필터링이 초기화됨
2. 테이블에 10개 이상 행이 있을 때 체크박스 선택 오작동
3. 모달 내 우편번호 자동 변환이 작동하지 않음
버그 원인을 분석하고 수정 방안을 제시해주세요.
