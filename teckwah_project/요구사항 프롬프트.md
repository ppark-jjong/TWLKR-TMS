# 대시보드 (Dashboard) 기능 상세 명세

## 1. API 목록
- GET /dashboard/list (대시보드 목록 조회)
- POST /dashboard (대시보드 생성)
- GET /dashboard/{dashboard_id} (상세 정보 조회)
- PATCH /dashboard/{dashboard_id}/status (상태 변경)
- PATCH /dashboard/{dashboard_id}/remark (메모 수정)
- POST /dashboard/assign (배차 처리)
- GET /dashboard/date-range (조회 가능 날짜 범위)

## 2. Frontend UI/UX 상세

### 메인 레이아웃
- 좌측 사이드바 고정, 우측 콘텐츠 영역 스크롤
- 상단 영역 고정 (검색/필터/버튼)
- 테이블 영역 스크롤 (calc(100vh - 상단높이))

### 상단 컨트롤 영역
1. 왼쪽 그룹
- DatePicker: 사이즈 large, 너비 280px
- 날짜 범위 텍스트: secondary 색상, Body-Medium 폰트

2. 오른쪽 그룹 (Space 컴포넌트, gap 16px)
- 생성 버튼: type="primary", PlusOutlined 아이콘
- 배차 버튼: default type, CarOutlined 아이콘
- 새로고침 버튼: ghost type, ReloadOutlined 아이콘

### 테이블 구성
- 체크박스 컬럼: width 60px
- 종류: width 80px, 중앙 정렬, 색상 구분
- 부서: width 80px, 중앙 정렬
- 출발 허브: width 100px, 중앙 정렬
- 배송 담당: width 100px, 중앙 정렬
- 주문번호: width 130px, 중앙 정렬
- 상태: width 100px, Tag 컴포넌트 사용
- ETA: width 150px, 중앙 정렬
- 도착 지역: width 130px, Tooltip 사용
- 행 스타일: 상태별 배경색, hover 효과
- 페이지네이션: 하단 고정, 50행/페이지

### 대시보드 생성 모달
- 너비: 1000px
- 2컬럼 레이아웃 (24 그리드 중 각 12)
- 모든 입력 필드는 large 사이즈
- 실시간 유효성 검증 피드백
- loading 상태 시 버튼 disabled

### 배차 모달
- 너비: 600px
- 선택된 주문 목록: 스크롤 박스 (최대 100px)
- 입력 필드 2개: 담당자명, 연락처
- 실시간 연락처 포맷팅
- confirm 시 loading 처리

### 상세 정보 모달
- 너비: 1200px
- 3개 섹션으로 구분 (기본정보/배송정보/메모)
- 정보 표시: Card 컴포넌트 활용
- 상태 변경: 드롭다운 + 저장 버튼
- 메모 수정: TextArea + 저장 버튼

## 3. Backend 서비스 동작

### DashboardService 주요 기능
1. 목록 조회 처리
- ETA 인덱스 활용 쿼리 최적화
- 데이터 정렬: ETA 컬럼 기준 ASC
- 날짜 범위 로직: KST 기준 처리

2. 상태 관리 로직
- 상태 변경 시 시간 자동 업데이트:
 * WAITING → IN_PROGRESS: depart_time = 현재시간
 * IN_PROGRESS → COMPLETE/ISSUE: complete_time = 현재시간
 * CANCEL/WAITING으로 변경: 시간 필드 모두 NULL
- 상태 변경 규칙 검증:
 * 현재 상태 기준 가능한 다음 상태 확인
 * 배차 정보 존재 여부 확인
 * 권한 레벨 확인

3. 배차 처리 로직
- 대상 건들의 상태 일괄 검증 (WAITING)
- 벌크 업데이트 처리
- 트랜잭션 관리

4. 우편번호 처리
- postal_code 테이블 조인
- 트리거를 통한 자동 정보 업데이트:
 * 도시/지역 정보
 * 거리 계산
 * 예상 소요 시간
- db 쿼리 트리거를 이용하여 절대 오류가 없어야함함

5. 데이터 검증
- 필수 필드 존재 여부
- 데이터 타입 검증
- 비즈니스 규칙 검증:
 * ETA는 현재 시점 이후
 * 연락처 형식 확인
 * 우편번호 존재 여부

## 4. 에러 처리 및 메시지
1. Frontend 메시지 처리
- 생성 성공: "대시보드가 생성되었습니다"
- 배차 성공: "배차가 완료되었습니다"
- 상태 변경: "{상태}로 변경되었습니다"
- 메모 수정: "메모가 업데이트되었습니다"

2. Backend 에러 처리
- 상태 변경 제한: "{현재상태}에서 {변경상태}로 변경할 수 없습니다"
- 배차 제한: "대기 상태가 아닌 주문이 포함되어 있습니다"
- DB 오류: "데이터 처리 중 오류가 발생했습니다"

# 관리자 페이지 (AdminPage) 기능 상세 명세

## 1. API 목록
- GET /dashboard/admin/list (기간별 목록 조회)
- DELETE /dashboard (선택 항목 삭제)
- 기타 API는 Dashboard와 공유

## 2. Frontend UI/UX 상세

### 메인 레이아웃 (DashboardPage와 차이점)
- RangePicker 사용 (DatePicker 대신)
- 삭제 버튼 추가 (DeleteOutlined 아이콘)

### 기간 선택 영역
- RangePicker: 사이즈 large, 너비 360px
- 기본값: 7일 전 ~ 오늘
- allowClear: false

### 테이블 기능
- DashboardPage와 동일한 컴포넌트 사용
- isAdminPage prop으로 관리자 기능 활성화
- 삭제 버튼: danger type, 선택된 행 있을 때 활성화

### 상세 정보 모달
- DashboardPage와 동일한 구조
- 상태 변경 시 제한 없음 (is_admin: true)
- 다만 상태 변경 시 한번 더 물어보는 모달창 필요 (정말 변경하시겠습니까?)

## 3. Backend 서비스 동작

### AdminDashboardService 특화 기능
1. 권한 검증
- user_role === 'ADMIN' 확인
- 권한 없을 시 403 Forbidden 응답

2. 기간별 조회 처리
- start_date, end_date 사이의 전체 ETA 컬럼 기준 데이터
- 상태 무관 조회
- 페이지네이션은 프론트엔드 처리

3. 삭제 처리
- 선택된 ID 목록 벌크 삭제
- 상태 제한 없음
- 트랜잭션 처리

2. 차트 구성
- X축: 시간대 (08-22시 1시간 단위, 22-08시 통합)
- Y축: 접수 건수
- 그룹화: 부서별 3개 막대
- 평균선 표시

3. 통계 카드
- 총 접수량
- 주간/야간 비율
- 시간당 평균
